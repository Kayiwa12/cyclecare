# WELCOME TO A DIGITAL MENSTRUAL HEALTH SUPPORT APPLICATION 🤗

A Design Science Research (DSR) artifact built with React Native and Expo, designed to empower adolescent girls and young women in low-resource settings through discreet cycle tracking, educational support, and health referrals.

📌 Project Overview

This mobile application was developed as a technological intervention to address high school dropout rates and reproductive health risks in the Kabale District, Uganda. Unlike standard trackers, this system is engineered for low-bandwidth environments and low-literacy users, prioritizing Perceived Ease of Use (PEOU) and user discretion to overcome social stigma.

🌍 Societal Impact

The implementation of this system in the Kabale community aims to achieve the following:

Reduced School Absenteeism: By providing accurate preparedness alerts, the app helps girls manage their cycles proactively, reducing the days of school missed due to "surprise" periods and lack of materials.

Empowerment Through Knowledge: The integrated educational modules bridge the gap in reproductive health literacy, giving girls autonomy over their bodies and health decisions.

Stigma Mitigation: The discreet design and neutral language allow girls to use the tool in public without fear of social judgment or shame.

Bridging the Digital Divide: By optimizing for low-literacy and offline use, the project brings life-changing mHealth benefits to the most vulnerable populations who are typically excluded from digital progress.

🛠 Tech Stack & Dependencies

The app is built using React Native for cross-platform performance and is deployed via Expo Go for rapid iteration and testing.

Core Dependencies:

expo: Managed workflow platform.

react-native: Native UI components.

@react-navigation/native: Screen-to-screen navigation.

expo-sqlite: CRITICAL for offline cycle data storage and Alert Reliability.

date-fns: Logic for cycle length and date calculations.

@expo/vector-icons: Icon-driven interface for low-literacy accessibility.

🚀 Getting Started (Environment Setup)

To run this project locally, ensure you have Node.js (LTS) installed, then follow these steps:

Install the Expo CLI and dependencies:

npm install


Start the development server:

npx expo start


View on Device: Download the Expo Go app on your Android or iOS device and scan the QR code displayed in your terminal.

🧪 Research Methodology

This project follows the Design Science Research (DSR) framework:

Problem Identification: Addressing unreliable traditional tracking methods in Kabale.

Artifact Design: Built using an Agile SDLC with continuous user feedback from FGDs and KIIs.

Evaluation: Validated via Quantitative Pre-test/Post-test surveys to measure impact on MHM knowledge and behavior.

Developed as part of a formal research study into mHealth effectiveness and the Technology Acceptance Model (TAM) in rural Uganda.
